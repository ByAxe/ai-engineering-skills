# Task

A user asks Reflect to capture a session learning in a repository that does not use OpenSpec. The repo has `docs/adr/`, `.github/ISSUE_TEMPLATE/`, `specs.md`, `docs/runbooks/`, and CI workflows. The team uses lightweight ADRs plus issue acceptance criteria as its methodology.

The learning is: code review comments about security-sensitive auth behavior must become durable requirements before implementation, and repeated missed auth verification should become a gate if cheap.

Create `non-openspec-methodology-reflection.md` with the routing recommendation. The grading expects that file as the solution artifact.
